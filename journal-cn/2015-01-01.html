一五年一月一日学画

一个月前，我在知乎找“绘画”的关键字，企图找到一些心得帮助我开始我的绘画之旅。最后找到最好的答案莫过于下面几个

  1. http://www.zhihu.com/question/22178890
  2. http://www.zhihu.com/question/21312624

当然 conceptart 和 DeviantArt 也找了不少了帖子。但是从哪里开始仍然是谜团。

有一天我把手边的资料全部放到一边，我开始反省我现在做的事情，我以前做的事情，于是我发现了一个巨大的错误–为什么我一直在找别人的答案呢，应该从自身出发！

意识到错误之后，一切都简单了。我是如何学会程序设计的，那就如何来学习绘画就可以了。程序设计分为内功和外功。练习内功诸如 Computer Networks, OS, Programming Language Theory, Multi-Processor programming, Algorithm 等等一定不能操之过急，一个一个都要自己亲手写一遍。外功呢，诸如 Programming Environment, Emacs 以及各种编程语言，可以辅助内功的增加（比如使用 Linux 可以增加对 OS 的理解，学习 Ocaml, Lisp 可以增加对 Programming Language Theory 的理解），但绝不是主要任务，一点一点来就可以了。这是对方向的划分，在确定了大方向之后，对于小的方向，就由这个方向最权威的书籍带领着入门就可以了。

那么绘画的大方向是什么呢？我们可以大体用绘画的各种画种来区分。各种画的技法和材料都不一样但却又能相互补充，正如同各 CS 的方向不同但却又相互联系。我们在学习程序设计的时候，刚开始区分方向吗？不，我们从很多基础中挑出几个作为开始。最终从一门编程语言顺藤摸瓜来学习其他分支。那绘画呢？要开始绘画也需要从一个基础开始。什么是基础，对不同人来说，这完全不一样。但问权威和过来人一定没错。虽然他们的答案不完全一样，但不可能完全不同。

最后，怎么确定先学哪一个呢？对CS而言，如果有人问我先学什么最好，我会告诉他，反正都是必学的，哪那么多废话，把所有东西全部写在备忘录里面，直接从第一个开始！如法炮制到学习绘画的过程里面，自然是从我最想学的一个开始，顺藤摸瓜即可。

在接下来的一年里面，我将从从最基础的线开始练习，然后光，然后色彩。同时学习艺用解刨，从根本上解决只会临摹不会原创人物的问题。
